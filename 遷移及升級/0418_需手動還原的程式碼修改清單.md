### 2. Socket.IO 連線修正（Frappe 原始碼）

**參考文件：** `SocketIO連線修復說明.md`

**修改檔案 1：**
`/home/stanley/frappe-bench/apps/frappe/frappe/realtime/middlewares/authenticate.js`

**修改說明：**
從 nginx reverse proxy 進來的連線，`origin` 與 `host` 不一致導致「Invalid origin」錯誤。加入信任代理檢查，當有 `X-Frappe-Site-Name` header 時跳過 origin 驗證。

```javascript
// 在 get_hostname 比對之前加入
const via_proxy = !!socket.request.headers["x-frappe-site-name"];
if (!via_proxy && get_hostname(socket.request.headers.host) != get_hostname(socket.request.headers.origin)) {
    next(new Error("Invalid origin"));
    return;
}
```

**修改檔案 2：**
`/home/stanley/frappe-bench/apps/frappe/frappe/realtime/utils.js`

**修改說明：**
`get_url()` 使用 `origin` header 作為 base URL，但同源請求 `origin` 為 undefined，導致 URL 解析失敗。加入 fallback 使用 `host` header。

```javascript
// 修改前
let url = socket.request.headers.origin;

// 修改後
let url = socket.request.headers.origin || `http://${socket.request.headers.host}`;
```

### 3. Nginx 設定（socket.io proxy）

**修改檔案：**
`/etc/nginx/sites-available/erpnext`

**修改說明：**
新增 `/socket.io` 的 proxy location，讓外部瀏覽器可透過 nginx 連接 socket.io server（port 9000），並設定 `X-Frappe-Site-Name` header 讓 Frappe 識別站台。

```nginx
location /socket.io {
    proxy_pass http://127.0.0.1:9000;
    proxy_http_version 1.1;
    proxy_set_header Upgrade $http_upgrade;
    proxy_set_header Connection "upgrade";
    proxy_set_header Host 192.168.1.60;
    proxy_set_header X-Real-IP $remote_addr;
    proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
    proxy_set_header X-Frappe-Site-Name site1.local;
    proxy_read_timeout 120;
}
```

套用指令：
```bash
sudo nginx -t && sudo systemctl reload nginx
```
