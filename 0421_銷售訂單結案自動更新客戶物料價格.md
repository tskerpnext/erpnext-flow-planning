# 銷售訂單結案後自動更新客戶物料價格

建立日期：2026-04-20
更新日期：2026-04-21
適用版本：ERPNext V16
發生環境：本機（192.168.1.60）

---

## 背景

ERPNext 預設不會在銷售訂單結案後自動更新價格表。目前遇到的問題：

- 客戶 126-油機，物料 SK-413-8P，SAL-ORD-2026-00003 實際成交價 6.6 元
- 系統 Standard Selling 價格表目前為 6.0 元（通用，無指定客戶）
- 新建訂單時預帶 6.0，需手動修正

---

## 銷售訂單取價順序

新建銷售訂單時，ERPNext 依以下優先順序取得物料單價：

1. **定價規則（Pricing Rule）** — 條件最精確的優先（指定客戶 + 指定物料 > 客戶群組 + 物料群組）
2. **客戶專屬 Item Price** — tabItem Price 中 customer 欄位指定該客戶的價格
3. **Standard Selling 價格表** — tabItem Price 中 customer 為空的通用價格
4. **物料成本價** — 上述都無時的備用（一般不用於售價）

> **注意：** 從報價單（Quotation）建立銷售訂單時，單價直接複製自報價單，不會重新查價格表。

---

## 報價單與銷售訂單的價格關係

| 建單方式 | 取價來源 |
|---------|---------|
| 直接新建銷售訂單 | 依取價順序查 Pricing Rule → Item Price → 價格表 |
| 從報價單建立銷售訂單 | 直接複製報價單的單價，不重新查價 |

因此本腳本（出貨更新 Item Price）的效果：
- **對直接新建的銷售訂單有效**：下次建單時自動帶入最新成交價
- **對從報價單轉來的訂單無直接影響**：報價單的價格需手動維護

---

## 需求

交貨單（Delivery Note）提交後，自動將出貨物料的成交單價更新至 `tabItem Price`，以客戶為單位記錄，供下次新建訂單時自動帶入最新成交價。

## 流程說明

```
交貨單提交（After Submit）
    ↓
讀取 doc.customer、doc.posting_date、doc.currency
    ↓
逐行讀取 doc.items
    ↓
每筆 item：item_code、rate、uom（跳過 rate = 0）
    ↓
查詢 tabItem Price：item_code + price_list + customer
  ├─ 有 → 更新 price_list_rate、valid_from、currency
  └─ 無 → 新增一筆客戶專屬 Item Price
```

---

## Server Script 規格

| 欄位 | 值 |
|------|-----|
| 腳本類型 | DocType Event |
| 參考文件類型 | Delivery Note（交貨單） |
| 事件 | After Submit |
| 腳本名稱 | auto-update-item-price-on-delivery |

> 使用 Delivery Note After Submit 觸發，出貨提交即更新。

---

## 完整腳本

```python
# 交貨單提交後，自動更新 tabItem Price（客戶專屬成交價）
# 觸發：Delivery Note After Submit

price_list = "Standard Selling"
customer = doc.customer
delivery_date = doc.posting_date
currency = doc.currency

for item in doc.items:
    if item.rate and item.rate != 0:
        existing = frappe.get_all(
            "Item Price",
            filters={
                "item_code": item.item_code,
                "price_list": price_list,
                "customer": customer,
            },
            fields=["name"],
            limit=1,
        )

        if existing:
            frappe.db.set_value("Item Price", existing[0].name, {
                "price_list_rate": item.rate,
                "valid_from": delivery_date,
                "currency": currency,
            })
        else:
            new_price = frappe.get_doc({
                "doctype": "Item Price",
                "item_code": item.item_code,
                "price_list": price_list,
                "customer": customer,
                "price_list_rate": item.rate,
                "valid_from": delivery_date,
                "selling": 1,
                "currency": currency,
                "uom": item.uom,
            })
            new_price.insert(ignore_permissions=True)
```

---

## 程式碼說明

| 段落 | 說明 |
|------|------|
| `item.rate == 0` | 跳過贈品或免費物料，不更新價格 |
| `frappe.get_all(...)` | 查詢是否已有此客戶 + 物料的 Item Price 紀錄 |
| `frappe.db.set_value` | 有紀錄則更新單價與生效日期 |
| `frappe.get_doc(...).insert` | 無紀錄則新增一筆客戶專屬 Item Price |
| `frappe.db.commit()` | Server Script 沙箱不支援，已移除，框架自動 commit |

## 相關 MariaDB 表格

### tabItem Price（價格表明細）
腳本的**寫入目標**，每筆記錄一個物料在特定價格表的售價。

| 欄位 | 說明 |
|------|------|
| name | 主鍵（自動產生） |
| item_code | 物料編號 |
| price_list | 價格表名稱（如 Standard Selling） |
| price_list_rate | 單價 |
| customer | 指定客戶（空白 = 通用，填入則為客戶專屬） |
| valid_from | 生效日期 |
| valid_upto | 失效日期（可空） |
| currency | 幣別 |
| uom | 計量單位 |
| selling | 是否為銷售用途（1 = 是） |

### tabSales Order Item（銷售訂單明細）
腳本的**讀取來源**，每筆記錄銷售訂單中一個物料行。

| 欄位 | 說明 |
|------|------|
| parent | 所屬銷售訂單編號（Sales Order name） |
| item_code | 物料編號 |
| rate | 實際成交單價（腳本取此值寫入 price_list_rate） |
| uom | 計量單位 |
| qty | 數量 |
| amount | 小計 |
