# 工單入庫依發料批號分拆說明書

建立日期：2026-04-28
適用版本：ERPNext V16
狀態：✅ 已部署測試通過

---

## 一、背景說明

工單發料（Material Transfer for Manufacture）時，若同一物料因批號庫存不足，需手動拆分成多筆領料（帶不同批號）。

但工單入庫（Manufacture 類型 Stock Entry）在建立時，系統依 BOM 重新帶入物料，不會繼承發料單的批號分拆設定，導致物料又回到一筆。

本功能透過 Client Script 自動讀取同工單的發料記錄，在入庫單建立時自動帶入批號分拆。

---

## 二、實際案例

| 單據 | 物料 | 情況 |
|------|------|------|
| 工單發料 MAT-STE-2026-00007 | SK-002-1 | 因批號庫存不足，手動拆成兩筆：批號A 8 個、批號B 22 個 |
| 工單入庫 MAT-STE-2026-00008 | SK-002-1 | 系統自動帶入為一筆 30 個，批號欄位空白 |

**期望結果：** 入庫單自動出現兩筆 SK-002-1，分別對應發料的批號與數量。

---

## 三、功能說明

### 觸發方式

| 觸發 | 時機 |
|------|------|
| **自動** | 建立 Manufacture 類型 Stock Entry 並設定工單時，延遲 1.5 秒自動執行 |
| **手動** | 點擊表單工具列的「**依發料批號分拆**」按鈕，可重複觸發 |

### 執行邏輯

```
1. 查詢同工單所有已過帳的「Material Transfer for Manufacture」
2. 讀取完整發料單（含 items 子表，權限跟隨父表）
3. 整理批號對應表：{ item_code → [{ batch_no, qty }] }
4. 重建入庫單物料行：
   - 成品行（is_finished_item）→ 不動
   - 無批號記錄的物料 → 不動
   - 有多批號的物料 → 依批號展開成多行
5. 以 frm.clear_table() + frm.add_child() 正確重建（避免殘留鬼魂列）
```

### 注意事項

- 發料單必須是**已過帳（Submit）**才會被讀取
- 若同一物料在多張發料單各有不同批號，系統**合併同批號數量**後再分拆
- 只更新批號（`batch_no`）與數量（`qty`、`transfer_qty`），不影響倉庫、成本等欄位

## 五、權限說明

Client Script 使用 `frappe.client.get` 讀取完整 Stock Entry 父單（含 items 子表），不需要 `Stock Entry Detail` 子表的獨立讀取權限。

一般製造使用者（Manufacturing User）只要有 Stock Entry 讀取權限即可正常使用。

## 六、Client Script 程式碼

**DocType：** `Stock Entry`　**類型：** Form

```javascript
frappe.ui.form.on('Stock Entry', {

    refresh: function(frm) {
        if (frm.doc.stock_entry_type === 'Manufacture' &&
            frm.doc.work_order &&
            frm.doc.docstatus === 0) {
            frm.add_custom_button(__('依發料批號分拆'), function() {
                apply_batch_split_from_issue(frm);
            });
        }
    },

    work_order: function(frm) {
        if (frm.doc.stock_entry_type === 'Manufacture' && frm.doc.work_order) {
            setTimeout(function() {
                apply_batch_split_from_issue(frm);
            }, 1500);
        }
    }

});

function apply_batch_split_from_issue(frm) {
    if (!frm.doc.work_order || frm.doc.stock_entry_type !== 'Manufacture') return;

    // Step 1: 取得同工單所有已過帳發料單名稱（查父表，使用者有權限）
    frappe.call({
        method: 'frappe.client.get_list',
        args: {
            doctype: 'Stock Entry',
            filters: {
                work_order: frm.doc.work_order,
                stock_entry_type: 'Material Transfer for Manufacture',
                docstatus: 1
            },
            fields: ['name'],
            limit_page_length: 50
        },
        callback: function(r) {
            if (!r.message || !r.message.length) {
                frappe.show_alert({ message: '找不到此工單已過帳的發料記錄', indicator: 'orange' });
                return;
            }

            // Step 2: 讀取完整發料單（含 items 子表，權限跟隨父表）
            let issue_names = r.message.map(d => d.name);
            let fetched = 0;
            let all_issue_items = [];

            issue_names.forEach(function(name) {
                frappe.call({
                    method: 'frappe.client.get',
                    args: { doctype: 'Stock Entry', name: name },
                    callback: function(resp) {
                        fetched++;
                        if (resp.message && resp.message.items) {
                            resp.message.items.forEach(function(item) {
                                if (!item.is_finished_item && item.batch_no) {
                                    all_issue_items.push(item);
                                }
                            });
                        }
                        if (fetched === issue_names.length) {
                            process_batch_split(frm, all_issue_items);
                        }
                    }
                });
            });
        }
    });
}

function process_batch_split(frm, issue_items) {
    if (!issue_items.length) {
        frappe.show_alert({ message: '發料記錄中無批號資訊', indicator: 'orange' });
        return;
    }

    // 整理批號對應表 { item_code: [{batch_no, qty, transfer_qty}] }
    let batch_map = {};
    issue_items.forEach(function(row) {
        if (!batch_map[row.item_code]) batch_map[row.item_code] = [];
        let existing = batch_map[row.item_code].find(b => b.batch_no === row.batch_no);
        if (existing) {
            existing.qty += flt(row.qty);
            existing.transfer_qty += flt(row.transfer_qty);
        } else {
            batch_map[row.item_code].push({
                batch_no: row.batch_no,
                qty: flt(row.qty),
                transfer_qty: flt(row.transfer_qty) || flt(row.qty) * flt(row.conversion_factor || 1)
            });
        }
    });

    // 深拷貝現有物料行，避免 clear_table 後遺失資料
    let original_items = JSON.parse(JSON.stringify(frm.doc.items || []));

    // 用 clear_table 正確清除 Frappe 內部列記錄（避免殘留鬼魂列）
    frm.clear_table('items');

    // 逐列重建，需分拆的物料展開成多列
    let skip_fields = ['name', 'idx', '__islocal', '__unsaved', 'doctype', 'parent', 'parentfield', 'parenttype'];

    original_items.forEach(function(item) {
        let batches = batch_map[item.item_code];

        // 成品行 或 無批號記錄的物料 → 原樣加回
        if (item.is_finished_item || !batches || batches.length === 0) {
            let new_row = frm.add_child('items');
            Object.keys(item).forEach(function(k) {
                if (!skip_fields.includes(k)) new_row[k] = item[k];
            });
            return;
        }

        // 依批號分拆成多列
        batches.forEach(function(batch) {
            let new_row = frm.add_child('items');
            Object.keys(item).forEach(function(k) {
                if (!skip_fields.includes(k)) new_row[k] = item[k];
            });
            new_row.batch_no = batch.batch_no;
            new_row.qty = batch.qty;
            new_row.transfer_qty = batch.transfer_qty;
        });
    });

    frm.refresh_field('items');
    frm.dirty();

    frappe.show_alert({ message: '已依發料批號自動分拆物料行', indicator: 'green' });
}
```

## 七、已知限制

- 入庫數量依發料數量為準；若發料數量與 BOM 預計數量不符，以發料實際數量為主
- 不自動處理序號（Serial No），序號仍需手動填寫
