# 框架訂單與生產工單流程規劃

建立日期： 2026-04-18
適用版本： ERPNext v16（已驗證於 Frappe v16.15.0 / ERPNext v16.14.0）
發生環境： 客戶提供月度銷售預測（非正式訂單），未來將轉為正式訂單，需提前規劃生產

---

## 一、情境說明

客戶提供每月銷售預測（Forecast），非正式 Sales Order，但未來會逐批轉為正式訂單。
工廠需依此預測提前備料與生產，並能在系統中追蹤：

- 框架訂單（Blanket Order）與實際銷售訂單的關聯
- 分批生產的工單與框架訂單的關聯
- 已生產數量 vs 預測數量的進度控管

---

## 二、核心文件說明

| 文件 | 用途 |
|------|------|
| Blanket Order（框架訂單） | 記錄客戶預測總量與有效期間（位於 **Selling** 模組） |
| Sales Order（銷售訂單） | 客戶正式確認數量後建立，可從 BO 建立 |
| Production Plan（生產計劃） | 批次規劃多品項工單，可從 SO 或手動輸入 |
| Work Order（生產工單） | 實際生產指令，分批建立 |

---

## 三、標準流程（有正式訂單）

```
Blanket Order
    └─→ Sales Order（客戶確認部分數量後建立）
            └─→ Production Plan（Get Items From: Sales Order）
                    └─→ Work Order（分批）
```

**操作步驟：**
1. Selling → Blanket Order → 建立，填入客戶、品項、預測總量、有效期間
2. 客戶確認數量後，從 Blanket Order 建立 Sales Order
3. Manufacturing → Production Plan → Get Items From: Sales Order
4. 選取對應 Sales Order → Create Work Orders

---

## 四、預測性生產流程（提前生產，不等 Sales Order）

```
Blanket Order（參考預測數量）
    └─→ Production Plan（手動輸入品項與數量）
            └─→ Work Order（分批生產）
                    └─→ 成品入庫（Finished Goods Warehouse）
                            └─→ 客戶確認後建 Sales Order → 出貨
```

**操作步驟：**
1. Manufacturing → Production Plan → New
2. "Get Items From" 留空，手動在 Items 表格填入品項、數量（參考 BO 預測）
3. Save → Create Work Orders
4. 生產完成後成品入庫
5. 客戶正式下單 → 從 Blanket Order 建 Sales Order → 從成品倉出貨

---

## 五、自訂欄位：Work Order 關聯框架訂單

**目的：** 讓每張工單記錄是依哪張 Blanket Order 生產的

**設定步驟：**
1. 搜尋列輸入 **Customize Form** → DocType 選 **Work Order**
   > v16 無獨立 Setup 模組，建議直接用搜尋列（Ctrl+G）叫出 Customize Form
2. 在適當區段新增欄位：
   - **Label：** Blanket Order
   - **Fieldname：** blanket_order
   - **Type：** Link
   - **Options：** Blanket Order
3. 儲存

**使用方式：** 開立工單時，手動在此欄位選入對應的 Blanket Order 編號。

---

## 六、Connections 設定：Blanket Order 頁面顯示關聯工單

**目的：** 在 Blanket Order 頁面右側 Connections 區塊，直接看到所有關聯工單

**設定步驟：**
1. 搜尋列輸入 **Customize Form** → DocType 選 **Blanket Order**
2. 切換到 **Links** 分頁
3. 新增一筆：
   - **Document Type：** Work Order
   - **Link Fieldname：** blanket_order（即步驟五建立的自訂欄位名）
4. 儲存

**效果：** Blanket Order 頁面右側會顯示關聯工單數量，點擊可列出所有工單清單。

---

## 七、自訂報表：框架訂單生產進度

**目的：** 一眼查看各 BO 的預測數量、已開工單數量、已完工數量、剩餘數量

**建立路徑：** 搜尋列輸入 **Report** → New
> v16 建議直接用搜尋列（Ctrl+G）開啟 Report，或前往 Home → Reports → New

**設定：**
- Report Name: Blanket Order Production Status
- DocType: Blanket Order
- Type: Query Report

**SQL 語法範本：**

```sql
SELECT
    bo.name AS blanket_order,
    bo.customer,
    boi.item_code,
    boi.item_name,
    boi.qty AS forecast_qty,
    IFNULL(SUM(wo.qty), 0) AS planned_qty,
    IFNULL(SUM(wo.produced_qty), 0) AS produced_qty,
    boi.qty - IFNULL(SUM(wo.produced_qty), 0) AS remaining_qty
FROM
    `tabBlanket Order` bo
    JOIN `tabBlanket Order Item` boi ON boi.parent = bo.name
    LEFT JOIN `tabWork Order` wo ON wo.blanket_order = bo.name
        AND wo.production_item = boi.item_code
        AND wo.docstatus != 2
WHERE
    bo.docstatus = 1
GROUP BY
    bo.name, boi.item_code
ORDER BY
    bo.name, boi.item_code
```

**報表欄位說明：**

| 欄位 | 說明 |
|------|------|
| forecast_qty | BO 預測總數量 |
| planned_qty | 已建工單數量（含進行中） |
| produced_qty | 已實際完工數量 |
| remaining_qty | 尚未生產數量 |

---

## 八、銷售訂單與工單的關聯（標準流程）

若走「有 Sales Order 的流程」，工單與 SO 的關聯由系統自動建立：

- Work Order 內建 **Sales Order** 欄位（`sales_order`）
- 從 Production Plan 建立的工單，若 Plan 來自 SO，會自動帶入
- 可在 Sales Order 頁面的 Connections 看到關聯工單（原生支援，不需額外設定）

---

## 九、查詢方式總結

| 查詢需求 | 方式 |
|----------|------|
| 某 BO 下有哪些工單 | Blanket Order 頁面 Connections 區塊 |
| 某 BO 生產進度（數量） | Blanket Order Production Status 報表 |
| 快速篩選工單 | Work Order List → 篩選 Blanket Order = XXX |
| 某 SO 對應哪些工單 | Sales Order 頁面 Connections（原生） |

---

## 十、建議實施順序

1. 先設定 Work Order 自訂欄位（blanket_order）
2. 設定 Blanket Order 的 Connections Links
3. 建立 Query Report
4. 制定 SOP：每月初依 BO 預測建 Production Plan → 開工單 → 填入 BO 欄位
5. 客戶正式下單後，從 BO 建 Sales Order，出貨時從成品倉取貨
