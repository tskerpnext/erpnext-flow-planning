# 開票管理－會計功能說明

適用地區：中國大陸
系統版本：ERPNext / Frappe v16
更新日期：2026-04-23

---

## 功能總覽

ERPNext 開票管理中提供多項會計輔助功能，用於修正帳務異常或手動補記分錄。以下依功能逐一說明。

| 功能 | 用途 | 日常是否用到 |
|------|------|------------|
| 會計憑證更新台帳 | 重新產生 GL 借貸分錄 | 異常時才用 |
| 收付款台帳重過帳 | 重建付款台帳（PLE） | 異常時才用 |
| 自動核銷收付款 | 自動將收款配對到發票 | 預收款 / 批次對帳時用 |
| 取消收付款核銷 | 拆開錯誤的收款與發票配對 | 配對錯誤時用 |
| 日記帳憑證 | 手動建立會計分錄 | 期末調整 / 補記帳時用 |

---

## 一、會計憑證更新台帳

### 說明

銷售發票（或其他財務單據）**已提交**之後，若 GL 分錄出現異常或需要重新計算，可重新產生正確的 GL 分錄，而不需要取消發票再重新提交。

### 使用時機

| 情境 | 說明 |
|------|------|
| 匯率變動 | 外幣發票提交後，匯率設定有誤，需重新計算本幣金額 |
| 科目設定錯誤 | 發票提交時科目映射錯誤，修正科目設定後需重新過帳 |
| GL 分錄異常 | 系統錯誤導致 GL 分錄缺失或金額不符 |
| 會計期間調整 | 過帳期間設定問題，修正後重新更新分錄 |
| 稅費模板變更 | 發票提交後稅費科目有調整，需重新反映到 GL |

### 流程示意

```
正常情況：
銷售發票提交 ──自動──▶ 產生 GL 分錄（正確）

異常情況：
銷售發票提交 ──自動──▶ 產生 GL 分錄（有誤）
                                │
                                ▼
                    會計憑證更新台帳（重新過帳）
                                │
                                ▼
                        GL 分錄更正為正確值
```

### 注意事項

- 此操作不會取消或修改發票本身，只重新產生 GL 分錄
- 執行前確認科目設定與稅費模板已修正，再執行更新
- 若發票已有對應的收款憑證，更新後需確認應收帳款金額是否仍正確對應
- 更新後應核對增值稅科目（VAT）分錄是否正確反映 13% 或 6% 稅額

## 二、收付款台帳重過帳（Repost Payment Ledger）

### 說明

ERPNext v15 之後引入獨立的 **Payment Ledger Entry（PLE）** 表，專門追蹤發票與收款的對應關係及未收餘額。此功能用於重建 PLE，與 GL Entry 無關。

### 與會計憑證更新台帳的差異

| | 會計憑證更新台帳 | 收付款台帳重過帳 |
|---|---|---|
| 操作對象 | 總帳 GL Entry | 付款台帳 Payment Ledger Entry |
| 影響範圍 | 科目借貸分錄 | 發票與收款的對應關係、未收餘額 |
| 目的 | 修正帳科金額 | 修正應收應付餘額顯示 |

### 使用時機

| 情境 | 說明 |
|------|------|
| 應收餘額顯示異常 | 發票明明已收款，應收帳款報表仍顯示未付 |
| 資料庫直接操作後 | 直接刪除或修改 GL Entry / Payment Entry 後 PLE 未同步 |
| 系統升級後 | 從舊版升級到 v15/v16，PLE 尚未建立 |
| 資料移轉後 | 匯入歷史發票與收款資料，對應關係未正確建立 |
| 取消重新提交後 | 單據取消再提交，PLE 出現重複或遺漏 |

### 流程示意

```
異常情況（PLE 損壞）：
發票已收款  ──▶ GL 正確  ──▶ 但 PLE 未更新
                              發票仍顯示「未付款」
                              應收報表金額錯誤
                                     │
                                     ▼
                           收付款台帳重過帳
                                     │
                                     ▼
                              PLE 重新計算，報表恢復正確
```

### 注意事項

- 此操作不影響 GL Entry，純粹重建 PLE
- 重過帳後確認應收帳款（Accounts Receivable）報表餘額是否正確
- 日常操作正常時不需要執行，屬於修復工具
- 若頻繁需要重過帳，應排查根本原因（通常是直接操作資料庫所致）

## 三、自動核銷收付款（Auto Reconcile Payments）

### 說明

「核銷」是將**收款憑證**與**銷售發票**進行對沖配對，讓系統知道「哪筆錢對應哪張發票」，發票餘額才會歸零。自動核銷功能由系統依條件自動配對，無需逐筆手動操作。

### 何時收款與發票會「脫鉤」

正常從發票點「建立 > Payment」，系統會自動對應。以下情況會出現未核銷狀態：

| 情境 | 說明 |
|------|------|
| 直接建立 Payment Entry | 不從發票進入，而是直接新建收款，未指定發票 |
| 預收款 | 客戶先付訂金，發票之後才開 |
| 銀行對帳匯入 | 從銀行流水匯入付款記錄，未自動對應發票 |
| 收款金額跨多張發票 | 一筆收款對應多張發票，需手動或自動配對 |

### 自動核銷運作方式

```
未核銷的收款憑證
        +
未付款的銷售發票
        │
        ▼
自動核銷收付款
（系統依同一客戶、金額相符、日期範圍內自動配對）
        │
        ▼
發票狀態 ──▶ 已付款
PLE 對沖 ──▶ 餘額歸零
```

### 使用時機

| 時機 | 說明 |
|------|------|
| 預收款轉沖 | 客戶先付訂金，開票後執行自動核銷將訂金沖抵發票 |
| 批次對帳 | 月底一次性核銷當月所有收款與發票 |
| 銀行流水匯入後 | 匯入銀行對帳單後，讓系統自動配對付款與發票 |
| 手動收款未指定發票 | 財務直接建立收款憑證但未選發票，事後補對應 |

### 手動核銷入口

若自動核銷無法完全配對，可至：

> **Accounting > Payment Reconciliation**
> 選擇客戶 → 選未核銷收款 → 選未付發票 → 點「Reconcile」

## 四、取消收付款核銷（Unreconcile Payments）

### 說明

將已配對的收款憑證與發票的對應關係拆開，讓雙方回到未核銷狀態。**不取消收款憑證本身，也不取消發票。**

### 與取消收款憑證的差異

| | 取消核銷 | 取消收款憑證 |
|---|---|---|
| 收款憑證 | 保留，仍存在 | 取消，變為草稿 |
| 發票狀態 | 回到未付款 | 回到未付款 |
| GL Entry | 不變 | 反轉沖銷 |
| 適用情境 | 配對錯誤，需重新對應 | 收款本身有誤，需作廢 |

### 使用時機

| 情境 | 說明 |
|------|------|
| 核銷配對錯誤 | 收款對到錯誤的發票，需拆開重新配對 |
| 客戶要求變更 | 客戶要求將這筆款項改沖另一張發票 |
| 預收款沖錯發票 | 訂金核銷時選錯發票，需重新對應 |
| 部分核銷調整 | 金額拆分比例需要重新分配 |

### 操作流程

```
Accounting > Payment Reconciliation
        │
選擇客戶 ──▶ 查詢已核銷紀錄
        │
選取要拆開的配對紀錄
        │
點「取消核銷 / Unreconcile」
        │
        ▼
收款憑證 ──▶ 回到「未核銷」狀態
發票     ──▶ 回到「未付款」狀態
        │
        ▼
重新執行核銷，選擇正確的發票配對
```

### 注意事項

- 取消核銷後，應收帳款報表會重新出現該發票的未收餘額，屬正常現象
- 取消後需盡快重新核銷，避免報表期間出現誤差
- 若已做過銀行對帳（Bank Reconciliation），取消核銷不影響銀行帳戶餘額

## 五、日記帳憑證（Journal Entry）

### 說明

**手動建立會計分錄**的工具，不依附任何業務單據（發票、收款等），直接在會計科目之間做借貸記錄。

### 與業務單據的差異

| | 銷售發票 | 收款憑證 | 日記帳憑證 |
|---|---|---|---|
| 用途 | 記錄銷售業務 | 記錄收付款 | 手動補記任何分錄 |
| 自動產生 GL | 是 | 是 | 本身就是 GL |
| 需要業務單據 | 是 | 是 | 否 |
| 使用者 | 業務 / 財務 | 財務 | 會計師 |

### 使用時機

| 情境 | 說明 |
|------|------|
| 期末調整 | 計提費用、預提折舊、攤銷等 |
| 銀行手續費 | 銀行扣款無對應業務單據時直接入帳 |
| 壞帳沖銷 | 應收帳款確認無法收回，直接沖銷 |
| 期初餘額建立 | 系統上線時匯入歷史科目餘額 |
| 跨科目調整 | 錯帳修正、科目間轉帳 |
| 增值稅申報調整 | 進項稅額轉出、稅額調整分錄 |
| 股東往來 | 股東借款、還款等無對應單據的資金往來 |

### 中國大陸常見分錄

| 分錄類型 | 借方 | 貸方 |
|---|---|---|
| 計提增值稅 | 應交稅費－應交增值稅 | 應收帳款 |
| 銀行手續費 | 財務費用 | 銀行存款 |
| 壞帳準備 | 資產減值損失 | 壞帳準備 |
| 期末折舊 | 製造費用 / 管理費用 | 累計折舊 |
| 預收款轉收入 | 預收帳款 | 主營業務收入 |

### 操作流程

```
Accounting > Journal Entry > 新建
        │
選擇憑證類型（Journal Entry / Bank Entry / Cash Entry）
        │
逐行填寫：科目 ／ 借方金額 ／ 貸方金額
        │
確認借貸合計相等（借 = 貸）
        │
填寫備註（摘要）
        │
提交 ──▶ GL Entry 產生
```

### 注意事項

- 借貸金額必須平衡，否則無法提交
- 涉及增值稅的調整分錄需確認科目對應到正確的**應交稅費**明細科目
- 日記帳憑證提交後若需修改，須先取消再重新提交
- 建議在備註／摘要欄填寫清楚用途，方便日後查帳
