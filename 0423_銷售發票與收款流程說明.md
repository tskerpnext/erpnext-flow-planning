# 銷售發票與收款流程

適用地區：中國大陸  
系統版本：ERPNext / Frappe v16  
更新日期：2026-04-23

---

## 一、整體流程概覽

```
銷售訂單（Sales Order）
        │
        ▼
銷售發票（Sales Invoice）── 提交後 ──▶ 銷售訂單狀態變為「完成」
        │
        ▼
  確認款項到帳
        │
        ├─── 對公銀行轉帳（最常見）
        ├─── 支付寶企業版
        └─── 微信支付商戶
        │
        ▼
收款憑證（Payment Entry）── 提交後 ──▶ 發票狀態變為「已付款」
```

---

## 二、銷售發票（Sales Invoice）

### 建立步驟

1. 進入 **Selling > Sales Invoice > 新建**
2. 選擇客戶，系統自動帶入客戶資料
3. 填寫品項、數量、單價
4. 確認以下關鍵欄位：

| 欄位 | 說明 |
|------|------|
| **發票日期** | 開票日期，通常為今天 |
| **付款到期日（Due Date）** | 由付款條款自動計算；未設付款條款時預設為今天 |
| **付款條款（Payment Terms）** | 月结15天 / 月结30天 / 款到发货 等 |
| **增值稅稅率** | 依商品類型選擇 13%（貨物）或 6%（服務） |
| **公司銀行帳戶** | 客戶付款的目標對公帳戶 |

5. 確認無誤後點 **「提交」**

### 提交後效果

- 銷售訂單狀態 → **完成（Completed）**
- 應收帳款科目產生借方分錄
- 發票狀態顯示 **「未付款（Unpaid）」**

### 付款到期日說明

| 付款條款設定 | 到期日結果 |
|---|---|
| 未設付款條款 | 到期日 = 發票日期（即今天） |
| 款到发货 | 到期日 = 發票日期 |
| 月结15天 | 到期日 = 發票月份月底 + 15 天 |
| 月结30天 | 到期日 = 發票月份月底 + 30 天 |

> **建議**：在客戶主檔（Customer）設定好付款條款，開票時系統會自動帶入，無需每張手動填寫。

## 三、收款流程

### 3.1 收付款申請（Payment Request）

| 項目 | 說明 |
|------|------|
| **用途** | 系統發送付款通知給客戶 |
| **影響帳務？** | 否，不入帳 |
| **中國大陸適用情境** | 串接支付寶 / 微信支付商戶時使用 |
| **B2B對公轉帳** | 通常不需要此步驟 |

**操作路徑**（如有串接金流）：
> 銷售發票 → 建立 → Payment Request → 發送給客戶掃碼付款 → 系統自動產生收款憑證

---

### 3.2 收款憑證（Payment Entry）— 核心步驟

**此步驟為必做，款項才會正式入帳。**

#### 操作步驟

1. 開啟已提交的**銷售發票**
2. 點右上角 **「建立」→「Payment（付款）」**
3. 系統自動帶入客戶、金額、發票編號
4. 確認以下欄位：

| 欄位 | 填寫說明 |
|------|---------|
| **付款類型** | Receive（收款） |
| **付款日期** | 實際到帳日期（查銀行流水） |
| **付款方式** | 銀行轉帳 / 支付寶 / 微信支付 |
| **收款帳戶** | 款項進入的對公銀行帳戶 |
| **已收金額** | 核對實際到帳金額 |

5. 確認無誤後點 **「提交」**

#### 提交後效果

| 情況 | 發票狀態 |
|------|---------|
| 全額收款 | **已付款（Paid）** |
| 部分收款 | **部分付款（Partly Paid）** |

---

### 3.3 部分收款處理

客戶分批付款時：

1. 第一次到帳 → 建立第一筆收款憑證（填實際到帳金額）
2. 發票狀態變為「部分付款」，顯示剩餘未收餘額
3. 第二次到帳 → 再次從發票「建立 > Payment」，系統自動帶入剩餘金額
4. 全額收齊後狀態變為「已付款」

## 四、中國大陸收款方式對照

| 收款方式 | 適用場景 | ERPNext 操作 |
|---------|---------|-------------|
| **對公銀行轉帳** | B2B 企業間交易（最常見） | 確認銀行流水後手動建立收款憑證 |
| **支付寶企業版** | 電商、線上業務 | 可串接後自動產生；或手動建立 |
| **微信支付商戶** | 零售、線上業務 | 可串接後自動產生；或手動建立 |
| **承兌匯票** | 大型製造業延期付款 | 建立收款憑證時選擇對應付款方式 |

---

## 五、確認收款完成

**方式一：查看發票狀態**
> Selling > Sales Invoice → 確認狀態為「已付款」

**方式二：應收帳款報表**
> Accounting > Accounts Receivable → 確認該客戶發票不再出現在未付清單

**方式三：對帳銀行流水**
> Accounting > Bank Reconciliation → 將系統收款憑證與銀行流水逐筆核對

## 六、常見問題

**Q：發票提交後付款到期日顯示今天，正常嗎？**  
A：正常，代表未設付款條款。建議在客戶主檔設定「月结30天」等條款，之後開票自動帶入正確到期日。

**Q：客戶已轉帳，但發票還是顯示「未付款」？**  
A：需手動建立收款憑證（Payment Entry）並提交，發票狀態才會更新。

**Q：收款金額與發票金額不一致怎麼辦？**  
A：填寫實際到帳金額，提交後發票顯示「部分付款」，待客戶補款後再建立第二筆收款憑證。
