# 銷售訂單結案後自動更新客戶物料價格 — Server Script 規劃

建立日期：2026-04-20
適用版本：ERPNext V16
發生環境：本機（192.168.1.60）

---

## 需求說明

銷售訂單結案（狀態變為 Closed）時，自動將訂單中每個物料的成交單價，更新至 `tabItem Price`，以客戶為單位記錄，供下次新建訂單時自動帶入。

---

## 確認參數

| 項目 | 決定 |
|------|------|
| 觸發時機 | 銷售訂單 **結案（Close）** |
| 更新目標 | tabItem Price，price_list = Standard Selling |
| 取價邏輯 | 最新一張訂單，每個物料最後一筆成交價 |
| 有效日期 | valid_from = 訂單日期（transaction_date） |

---

## 流程說明

```
銷售訂單結案
    ↓
讀取 doc.customer、doc.transaction_date
    ↓
逐行讀取 doc.items
    ↓
每筆 item：item_code、rate、uom
    ↓
查詢 tabItem Price 是否有相同 item_code + price_list + customer
  ├─ 有 → 更新 price_list_rate、valid_from
  └─ 無 → 新增一筆
    ↓
儲存
```

---

## Server Script 規格

| 欄位 | 值 |
|------|-----|
| 腳本類型 | DocType Event |
| 參考文件類型 | Sales Order |
| 事件 | on_update（搭配狀態判斷 == "Closed"） |
| 語言 | Python |

> 說明：ERPNext Server Script 沒有「on_close」事件，改用 `on_update` 並判斷 `doc.status == "Closed"` 且前一狀態非 Closed，避免重複執行。

---

## 核心邏輯（虛擬碼）

```python
if doc.status != "Closed":
    return

price_list = "Standard Selling"
customer = doc.customer
order_date = doc.transaction_date

for item in doc.items:
    existing = frappe.get_all(
        "Item Price",
        filters={
            "item_code": item.item_code,
            "price_list": price_list,
            "customer": customer,
        },
        fields=["name"],
        limit=1,
    )

    if existing:
        frappe.db.set_value("Item Price", existing[0].name, {
            "price_list_rate": item.rate,
            "valid_from": order_date,
        })
    else:
        frappe.get_doc({
            "doctype": "Item Price",
            "item_code": item.item_code,
            "price_list": price_list,
            "customer": customer,
            "price_list_rate": item.rate,
            "valid_from": order_date,
            "selling": 1,
            "currency": doc.currency,
            "uom": item.uom,
        }).insert(ignore_permissions=True)

frappe.db.commit()
```

---

## 注意事項

1. **重複觸發防護**：on_update 每次存檔都觸發，需加狀態判斷，只在「剛變為 Closed」時執行一次（可用 `doc.get_doc_before_save()` 確認前一狀態）
2. **currency**：若多幣別，需確認 Item Price 的幣別與訂單一致
3. **uom**：若同物料有多種計量單位，price_list_rate 對應的 uom 需一致
4. **rate = 0 的情況**：若物料 rate 為 0（贈品等），需決定是否跳過不更新
