# ERPNext 銷售模組流程與月結對帳自動化規劃

日期：2026-04-22

---

## 一、銷售模組完整流程總覽

```
報價單 Quotation（可選）
  ↓
銷售訂單 Sales Order
  ↓
出貨單 Delivery Note（扣庫存）
  ↓
銷售發票 Sales Invoice（產生應收）
  ↓
收款憑單 Payment Entry（沖應收）
  ↓
✅ 結案
```

### 各階段說明

| 階段 | 文件類型 | 主要用途 |
|------|----------|----------|
| 1 | 報價單 Quotation | 對客戶提供報價，可直接轉銷售訂單 |
| 2 | 銷售訂單 Sales Order | 確認品項、數量、價格、交期 |
| 3 | 出貨單 Delivery Note | 記錄實際出貨，自動扣減庫存 |
| 4 | 銷售發票 Sales Invoice | 正式請款，產生應收帳款分錄 |
| 5 | 收款憑單 Payment Entry | 記錄客戶付款，沖銷應收帳款 |

> 若不需管理庫存，可跳過出貨單，直接從銷售訂單開發票。

## 二、銷售發票合併多張訂單開立

### 方法 A：在發票內拉入多筆出貨單
1. 新增「銷售發票 Sales Invoice」
2. 選好客戶（Customer）
3. 品項列表下方點「Get Items From」→ 選 Delivery Note 或 Sales Order
4. 勾選多張，點確認，品項自動合併帶入

### 方法 B：從出貨單列表批次建立
- 出貨單列表 → 篩選客戶 + 日期區間 → 勾選多筆 → 點「建立銷售發票」

> **注意**：多張訂單必須是同一個客戶，幣別與付款條件建議一致。

---

## 三、先對帳再開發票的作業方式

### 方法一：草稿發票（最常用）
1. 建立銷售發票後「儲存」但不「提交」
2. 列印草稿寄給客戶確認
3. 客戶確認無誤後再「提交」正式入帳

> 草稿狀態不產生應收分錄，不影響帳務。

### 方法二：Delivered Items to be Billed 報表
- 路徑：會計 → 報表 → Delivered Items to be Billed
- 篩選客戶 + 日期區間，顯示已出貨但未開票品項
- 列印或截圖給客戶確認後再批次開票

### 方法三：Statement of Accounts（已有發票時）
- 路徑：會計 → 報表 → Statement of Accounts
- 可選客戶、日期區間，列出發票、付款、餘額
- 支援直接 Email 給客戶

## 四、月結對帳月份管理

### 問題背景
用出貨日期篩選會因跨月出貨導致歸期錯誤，需要額外欄位標記每筆出貨屬於哪個對帳期。

### 解法：自訂「對帳月份」欄位

在 **出貨單（Delivery Note）** 新增自訂欄位：

| 設定項目 | 值 |
|----------|----|
| Document Type | Delivery Note |
| Label | 對帳月份 |
| Field Name | reconciliation_month |
| Field Type | Data |
| In List View | ✅ |
| In Standard Filter | ✅ |

格式範例：`2026-03`、`2026-04`

### 使用流程
1. 出貨時填入「對帳月份」
2. 月底依「對帳月份」篩選出貨單 → 列印給客戶確認
3. 客戶確認後勾選該月出貨單 → 批次建立銷售發票

## 五、客戶結帳截止日管理

### 問題背景
不同客戶有不同結帳截止日（15號、20號、月底），需從客戶源頭設定，讓系統自動判斷出貨歸屬期。

### ERPNext 原生功能限制
- **付款條件 Payment Terms**：可設在客戶主檔，但處理的是「付款到期日」，非「對帳截止日」
- **無原生「結帳截止日」欄位**，需自訂

### 解法：在客戶主檔加自訂欄位

在 **客戶（Customer）** 新增自訂欄位：

| 設定項目 | 值 |
|----------|----|
| Document Type | Customer |
| Label | 結帳截止日 |
| Field Name | billing_cutoff_day |
| Field Type | Int（整數） |

填入範例：`15`、`20`、`31`（月底）

## 六、自動填入「對帳月份」Server Script

出貨單儲存時，自動讀取客戶的結帳截止日，計算並填入對帳月份欄位。

### 設定步驟

1. 路徑：設定 → 開發人員 → Server Script → 新增
2. 填入以下設定：

| 設定項目 | 值 |
|----------|----|
| Script Type | DocType Event |
| Reference DocType | Delivery Note |
| DocType Event | Before Save |

### 腳本內容

```python
from datetime import date

# 取得客戶的結帳截止日
cutoff = frappe.db.get_value(
    "Customer", doc.customer, "billing_cutoff_day"
)

if cutoff and doc.posting_date:
    d = doc.posting_date  # 出貨日期
    day = d.day
    year = d.year
    month = d.month

    # 若出貨日 超過 截止日 → 算下個月對帳
    if day > int(cutoff):
        if month == 12:
            month = 1
            year += 1
        else:
            month += 1

    # 設定對帳月份格式 YYYY-MM
    doc.reconciliation_month = f"{year}-{month:02d}"
```

### 邏輯說明

| 客戶結帳截止日 | 出貨日 | 對帳月份歸屬 |
|----------------|--------|--------------|
| 20號 | 4/10（≤ 20） | 2026-04 |
| 20號 | 4/25（> 20） | 2026-05 |
| 15號 | 4/10（≤ 15） | 2026-04 |
| 15號 | 4/20（> 15） | 2026-05 |
| 31號（月底） | 任意日 | 永遠歸當月 |

### 注意事項
- 需開啟開發者模式：設定 → 系統設定 → 勾選 Developer Mode
- 客戶未設截止日時，欄位保留原值不覆寫
