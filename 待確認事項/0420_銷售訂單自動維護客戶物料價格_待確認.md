# 銷售訂單自動維護客戶物料價格 — 待確認事項

建立日期：2026-04-20
適用版本：ERPNext V16
發生環境：本機（192.168.1.60）

---

## 背景

ERPNext 預設不會在銷售訂單結案後自動更新價格表。目前遇到的問題：

- 客戶 126-油機，物料 SK-413-8P，SAL-ORD-2026-00003 實際成交價 6.6 元
- 系統 Standard Selling 價格表目前為 6.0 元（通用，無指定客戶）
- 新建訂單時預帶 6.0，需手動修正

---

## 需求

銷售訂單 提交（Submit）或 結案（Close）後，自動將成交單價更新回對應客戶的物料價格，使下次新建訂單可自動帶入最新成交價。

---

## 待確認事項

| # | 問題 | 選項 | 確認結果 |
|---|------|------|----------|
| 1 | 觸發時機 | A. 提交（Submit） / B. 結案（Close） | |
| 2 | 更新哪張價格表 | A. Standard Selling（通用） / B. 另建「客戶成交價」專用價格表 | |
| 3 | 同客戶同物料多張訂單時，取哪個價格 | A. 最新一張訂單的價格 / B. 其他邏輯 | |
| 4 | 是否記錄有效日期 | A. valid_from = 訂單日期 / B. 不設有效期 | |

---

## 確認後執行方向

以 Server Script 實現，觸發後逐行讀取訂單 items，依 item_code + customer 查找或新增 tabItem Price，更新 price_list_rate。

---

## 相關資料

- 目前 SK-413-8P 價格表紀錄：Standard Selling / 6.0 元 / 無指定客戶 / valid_from 2026-04-16
- 相關 MariaDB 表：`tabItem Price`（item_code, price_list, price_list_rate, customer, valid_from）
