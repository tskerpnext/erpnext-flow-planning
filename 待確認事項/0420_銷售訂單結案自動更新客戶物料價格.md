# 銷售訂單結案後自動更新客戶物料價格

建立日期：2026-04-20
適用版本：ERPNext V16
發生環境：本機（192.168.1.60）

---

## 背景

ERPNext 預設不會在銷售訂單結案後自動更新價格表。目前遇到的問題：

- 客戶 126-油機，物料 SK-413-8P，SAL-ORD-2026-00003 實際成交價 6.6 元
- 系統 Standard Selling 價格表目前為 6.0 元（通用，無指定客戶）
- 新建訂單時預帶 6.0，需手動修正

---

## 需求

銷售訂單結案（Close）後，自動將訂單中每個物料的成交單價更新至 `tabItem Price`，以客戶為單位記錄，供下次新建訂單時自動帶入最新成交價。

---

## 確認參數

| # | 問題 | 確認結果 |
|---|------|----------|
| 1 | 觸發時機 | ✅ 結案（Close） |
| 2 | 更新哪張價格表 | ✅ Standard Selling（tabItem Price） |
| 3 | 同客戶同物料多張訂單取價邏輯 | ✅ 最新一張，每個物料最後一筆成交價 |
| 4 | 有效日期 | ✅ valid_from = 訂單日期（transaction_date） |

---

## 流程說明

```
銷售訂單結案
    ↓
讀取 doc.customer、doc.transaction_date
    ↓
逐行讀取 doc.items
    ↓
每筆 item：item_code、rate、uom
    ↓
查詢 tabItem Price 是否有相同 item_code + price_list + customer
  ├─ 有 → 更新 price_list_rate、valid_from
  └─ 無 → 新增一筆
    ↓
儲存
```

---

## Server Script 規格

| 欄位 | 值 |
|------|-----|
| 腳本類型 | DocType Event |
| 參考文件類型 | Delivery Note（交貨單） |
| 事件 | After Submit |
| 腳本名稱 | auto-update-item-price-on-delivery |

> 使用 Delivery Note After Submit 觸發，出貨提交即更新，不需等到結案。

---

## 完整腳本

```python
# 交貨單提交後，自動更新 tabItem Price（客戶專屬成交價）
# 觸發：Delivery Note After Submit

price_list = "Standard Selling"
customer = doc.customer
delivery_date = doc.posting_date
currency = doc.currency

for item in doc.items:
    if item.rate and item.rate != 0:
        existing = frappe.get_all(
            "Item Price",
            filters={
                "item_code": item.item_code,
                "price_list": price_list,
                "customer": customer,
            },
            fields=["name"],
            limit=1,
        )

        if existing:
            frappe.db.set_value("Item Price", existing[0].name, {
                "price_list_rate": item.rate,
                "valid_from": delivery_date,
                "currency": currency,
            })
        else:
            new_price = frappe.get_doc({
                "doctype": "Item Price",
                "item_code": item.item_code,
                "price_list": price_list,
                "customer": customer,
                "price_list_rate": item.rate,
                "valid_from": delivery_date,
                "selling": 1,
                "currency": currency,
                "uom": item.uom,
            })
            new_price.insert(ignore_permissions=True)

frappe.db.commit()
```

---

## 程式碼說明

| 段落 | 說明 |
|------|------|
| `doc.status != "Closed"` | 非結案狀態直接跳出，不執行任何更新 |
| `get_doc_before_save()` | 取存檔前的狀態，防止結案後再次編輯時重複觸發 |
| `item.rate == 0` | 跳過贈品或免費物料，不更新價格 |
| `frappe.get_all(...)` | 查詢是否已有此客戶 + 物料的 Item Price 紀錄 |
| `frappe.db.set_value` | 有紀錄則更新單價與生效日期 |
| `frappe.get_doc(...).insert` | 無紀錄則新增一筆客戶專屬 Item Price |
| `frappe.db.commit()` | 確保所有變更寫入資料庫 |

## 相關 MariaDB 表格

### tabItem Price（價格表明細）
腳本的**寫入目標**，每筆記錄一個物料在特定價格表的售價。

| 欄位 | 說明 |
|------|------|
| name | 主鍵（自動產生） |
| item_code | 物料編號 |
| price_list | 價格表名稱（如 Standard Selling） |
| price_list_rate | 單價 |
| customer | 指定客戶（空白 = 通用，填入則為客戶專屬） |
| valid_from | 生效日期 |
| valid_upto | 失效日期（可空） |
| currency | 幣別 |
| uom | 計量單位 |
| selling | 是否為銷售用途（1 = 是） |

### tabSales Order Item（銷售訂單明細）
腳本的**讀取來源**，每筆記錄銷售訂單中一個物料行。

| 欄位 | 說明 |
|------|------|
| parent | 所屬銷售訂單編號（Sales Order name） |
| item_code | 物料編號 |
| rate | 實際成交單價（腳本取此值寫入 price_list_rate） |
| uom | 計量單位 |
| qty | 數量 |
| amount | 小計 |
