# ERPNext v16 導入規劃時程

**建立日期**：2026-04-30
**更新日期**：2026-04-30
**適用版本**：ERPNext v16（Frappe v16）
**適用情境**：電纜產品 + 加工產品 製造業，全廠約 140 人，兩條產品線各約 65 人

---

## 一、專案概述

### 1.1 導入範圍

| 模組 | 說明 |
|------|------|
| 業務模組（Selling） | 報價、銷售訂單、框架訂單、出貨、銷售發票 |
| 採購模組（Purchasing） | 請購、採購訂單、收料、採購發票 |
| 生產模組（Manufacturing） | BOM、生產計劃、生產工單、發料、入庫（**不報工**） |
| 庫存模組（Stock） | 倉庫管理、庫存調整、物料批號、庫存盤點 |
| 財務模組（Accounts） | 應收帳款、應付帳款、增值稅、對帳、付款管理 |

### 1.2 核心目標

1. **帳料符合現況**：庫存帳務與實際庫位數量一致，批號可追溯
2. **生產不報工**：生產工單以「發料 → 成品入庫」為主線，不記錄工序工時
3. **列印格式完備**：涵蓋業務、採購、生產、財務共約 20 種格式
4. **兩條產品線並行**：電纜產品（电缆产品）與加工產品（加工产品）各有獨立倉庫、BOM 架構與流程

### 1.3 公司規模

| 項目 | 數量 |
|------|------|
| 全廠員工 | 約 140 人 |
| 電纜產品線 | 約 65 人 |
| 加工產品線（線束） | 約 65 人 |
| 管理/財務/業務 | 約 10 人 |

## 四、Phase 1：業務訪談與需求分析（第 3～8 週）

### 4.1 訪談目標

透過結構性訪談，確認：
1. 各部門目前作業方式（現況 As-Is）
2. 期望 ERP 協助改善的項目（目標 To-Be）
3. 各模組主要使用者與決策層級
4. 現有表單格式（作為列印格式設計依據）

### 4.2 訪談對象與重點

#### A. 業務部門

| 訪談主題 | 關鍵問題 |
|----------|----------|
| 客戶管理 | 客戶分類？信用條件？價格體系（含客戶物料價）？ |
| 報價流程 | 報價單格式？BOM 成本是否顯示於報價？折扣方式？ |
| 訂單管理 | 框架訂單 vs 正式訂單的比例？客戶採購單號如何管理？ |
| 出貨流程 | 是否需揀貨單？多張訂單合批出貨？出貨通知格式？ |
| 開票流程 | 增值稅專用發票格式？月結截止日？合規要求？ |
| 對帳流程 | 客戶對帳單格式？月結日是否固定？如何處理退貨？ |

#### B. 採購部門

| 請購流程 | 誰可以申請請購？審批層級？緊急採購流程？ |
| 採購訂單 | 長期採購合約（框架採購單）是否有需求？ |
| 收料流程 | 收料是否需要品質檢驗紀錄？批號如何產生？ |

#### C. 生產部門（電纜產品線）

| 訪談主題 | 關鍵問題 |

| 發料流程 | 領料單位是誰？從哪個倉庫領？需要批號追蹤嗎？ |

#### D. 生產部門（加工產品線/線束）

### 5.3 帳料符合現況設計原則

帳料一致性是本次導入的核心要求，以下為設計準則：

#### 庫存精度措施

| 措施 | 做法 |
|------|------|
| 每筆進出貨即時過帳 | 禁止月底補單，強制使用者即時登錄 |
| 批號強制管理 | 原物料、成品均設定批號為必填 |
| 發料倉位對應 | 每個倉位對應唯一 ERP 倉庫，不混倉 |
| 盤點差異即時調整 | 月盤發現差異，次日以庫存調整單更正 |
| 物料標籤雙掃確認 | 收料與發料均需掃描物料批號標籤 |

#### 生產帳務設計（不報工）

```
生產工單流程（不報工版）
┌─────────────────────────────────────────────┐
│ 工單開立 → 發料（Material Transfer）         │
│         → 成品入庫（Manufacturing Entry）   │
│         → 工單關閉                          │
└─────────────────────────────────────────────┘

帳務分錄：
1. 發料：借 在制程仓（WIP） / 貸 物料仓（Raw Material）
2. 入庫：借 成品仓（FG）   / 貸 在制程仓（WIP）
3. 差異：物料消耗差異自動計入生產成本差異科目
```

#### 生產成本核算（簡化版）

| 成本項目 | ERPNext 處理方式 |
|----------|-----------------|
| 原物料成本 | 依發料數量 × 移動加權平均單價 |
| 人工成本 | 不報工 → 可選：月底分攤日記帳 或 暫不核算 |
| 製造費用 | 月底以日記帳分攤（選做） |

> **建議**：第一階段先僅追蹤原物料成本，人工與製造費用於上線 3 個月後再視需求加入。

## 六、Phase 3：資料移轉與教育訓練（第 21～26 週）

### 6.1 期初資料移轉

#### 資料準備順序（有依存關係）

```
1. 公司基礎設定（已在 Phase 0/1 完成）
2. 會計科目 (Chart of Accounts)
3. 稅費模板 (Tax Template)
4. 倉庫 (Warehouse)
5. 物料組 (Item Group)
6. 物料主檔 (Item Master)   ← 量最大，建議 Excel 批次匯入
7. 客戶 (Customer)          ← 含付款條件、稅務類型
8. 供應商 (Supplier)        ← 同上
9. 物料價格 (Item Price)    ← 銷售/採購單價
10. BOM                     ← 依賴物料主檔
11. 期初庫存 (Opening Stock) ← 最後匯入，以庫存調整單建立
12. 期初應收 / 應付         ← 以日記帳建立未結款項
```

#### 期初庫存匯入原則

- 匯入日期定為正式上線前一天（例：上線日 2026-11-01，期初庫存日期 2026-10-31）
- 所有物料需帶批號，批號格式依訪談結果決定（建議：`YYYYMM-序號`）
- 匯入後立即對帳：ERP 報表 vs 人工盤點清單，誤差率需 < 0.5%

### 6.2 教育訓練計劃

#### 訓練分組

| 組別 | 參與人員 | 訓練課時 | 訓練方式 |
|------|----------|----------|----------|
| 系統管理員 | IT 或指定管理人員（1-2人） | 16h | 一對一操作實習 |
| 業務操作 | 業務人員（預估 3-5人） | 12h | 分組課堂 + 實操 |
| 採購操作 | 採購人員（預估 2-4人） | 10h | 分組課堂 + 實操 |
| 生產計劃 | 計劃人員（預估 2-3人） | 8h | 分組課堂 + 實操 |
| 倉庫操作 | 倉管人員（預估 4-6人） | 10h | 分組實操為主 |
| 財務操作 | 財務人員（預估 2-4人） | 12h | 分組課堂 + 實操 |
| 管理查閱 | 主管層（預估 5-8人） | 4h | 報表查閱為主 |

#### 訓練課程大綱

**第一週：基礎操作與共用功能**
- ERP 登入、介面導覽
- 物料主檔查詢與基本操作
- 單據審批與提交原則
- 常見錯誤與解決方式

**第二週：各部門專項操作**
- 業務：報價→訂單→出貨→開票完整流程演練
- 採購：請購→詢價→採購→收料→付款演練
- 生產：BOM 查閱→工單建立→發料→入庫演練
- 倉庫：收料掃碼→標籤列印→發料確認→盤點操作
- 財務：對帳單生成→付款核銷→月結流程演練

**第三週：報表查閱與異常處理**
- 庫存報表、應收應付報表操作
- 帳料差異處理 SOP（庫存調整單）
- 無法刪除/取消的單據處理方式
- 備份與還原流程說明

## 七、Phase 4：平行試運行（第 27～30 週）

### 7.1 平行試運行說明

在正式上線前進行 4 週的「新舊系統並行」，目的是：
1. 確認 ERP 帳務與現有 Excel/手工帳一致
2. 讓使用者熟悉正式流程，降低上線恐慌
3. 發現並修正程式或設定問題

### 7.2 平行試運行規則

| 規則 | 說明 |
|------|------|
| 同步建單 | 新訂單同時在 ERP 和舊系統建立 |
| 每週對帳 | 每週五比對 ERP 庫存報表 vs 手工帳 |
| 差異登記 | 所有差異記錄在「差異追蹤表」，每週回顧 |
| 切換門檻 | 連續 2 週帳料差異 < 1%，且無重大系統問題 |

### 7.3 試運行檢核清單

- [ ] 完整走通業務→出貨→開票→收款流程（含退貨）
- [ ] 完整走通採購→收料→入庫→付款流程
- [ ] 電纜產品工單完整流程：BOM→工單→發料（原物料倉）→入庫
- [ ] 加工產品工單完整流程：BOM→線邊倉備料→工單→發料→入庫
- [ ] 月結流程走通：關帳→生成試算表→對帳
- [ ] 所有列印格式驗收：格式正確、資料完整
- [ ] 批號追蹤測試：選一批成品，追溯至原物料批號
- [ ] 帳料一致性：ERP 庫存帳 vs 實地盤點（抽查 3 個倉庫）

## 八、Phase 5：正式上線（第 31 週）

### 8.1 上線前最後確認

**T-7 天（上線前一週）**
- [ ] 正式機器環境最終確認（磁碟空間、備份策略）
- [ ] 使用者帳號全部設定完畢
- [ ] 期初資料最終版本匯入完成
- [ ] 所有列印格式在正式機驗收通過

**T-2 天**
- [ ] 向全廠發送上線通知（說明流程切換時間、緊急聯絡方式）
- [ ] 確認 Helpdesk 支援人員待命安排

**T-Day（上線日）**
- [ ] 舊系統停止新建單據（僅查閱）
- [ ] ERP 正式機對所有使用者開放
- [ ] 實施人員現場支援（電纜線、加工線各配一人）
- [ ] 財務人員確認期初餘額正確

### 8.2 上線後 24h 緊急支援

- 上線後 3 天，支援人員駐廠或隨時可遠端（微信立即回覆）
- 緊急問題定義：影響出貨、無法開立發票、庫存大幅異常
- 非緊急問題（列印格式微調、查詢方式不熟）納入正常改善流程

## 九、Phase 6：上線後檢討與改善（M8～M12）

### 9.1 月度檢討機制

| 週期 | 檢討內容 |
|------|----------|
| 上線後第 1 月 | 每週一次：問題清單 review、帳料差異分析、緊急調整 |
| 上線後第 2-3 月 | 每月一次：優化需求整理、報表品質評估 |
| 上線後第 3 月起 | 每月一次：新增功能規劃、使用深度評估 |

### 9.2 檢討評估指標（KPI）

| 指標 | 目標值 | 量測方式 |
|------|--------|----------|
| 帳料符合率 | ≥ 99% | 月盤差異金額 / 總庫存金額 |
| 單據及時過帳率 | ≥ 95% | 單據提交日期 vs 實際作業日期差 |
| 月結完成時效 | ≤ 月底後 5 個工作日 | 財務關帳日記錄 |
| 使用者滿意度 | ≥ 7/10 | 每月末問卷調查 |
| 列印格式正確率 | 100% | 使用者反映錯誤次數 |

### 9.3 常見上線後問題與因應

| 問題類型 | 常見原因 | 因應方式 |
|----------|----------|----------|
| 庫存帳料不符 | 未即時過帳、發料倉輸錯 | 加強 SOP 培訓、增加稽核頻率 |
| 列印格式資料缺失 | Jinja 模板欄位路徑錯誤 | 逐一確認欄位來源，修正模板 |
| 批號無法追蹤 | 發料時未選批號 | 設定批號為必填欄位，強制輸入 |
| 月結無法關帳 | 未提交的草稿單據存在 | 執行月底草稿清查 SOP |
| 生產工單成本異常 | BOM 用量與實際差異大 | 每月核對 BOM vs 實際用量，定期更新 |

### 9.4 改善需求排程

上線後收集的改善需求，依優先度排程：

| 優先度 | 定義 | 回應時效 |
|--------|------|----------|
| P1 緊急 | 影響正常營運（出不了貨/開不了票） | 24h 內修復 |
| P2 重要 | 影響工作效率，但有手動替代方案 | 1 週內規劃 |
| P3 優化 | 錦上添花，不影響核心流程 | 累積後每月統一處理 |

## 十、風險管理

| 風險項目 | 可能影響 | 因應措施 |
|----------|----------|----------|
| 期初資料品質差 | 上線後帳料不準確 | Phase 1 提前收集，Phase 3 嚴格驗收 |
| 使用者抗拒 | 流程推行困難 | 管理層明確支持、訓練期間充分溝通利益 |
| BOM 資料不完整 | 生產成本計算錯誤 | 生產主管負責 BOM 逐一確認與簽核 |
| ERPNext 版本升級衝突 | 自訂程式失效 | 升級前備份，升級後逐一驗證 Client Script |
| 增值稅申報不符合 | 稅務風險 | 財務主管在平行試運行階段核對每筆稅額 |
| 兩條產品線流程差異過大 | 系統難以統一設定 | Phase 1 深度訪談，必要時設不同 Print Format/流程分支 |

## 十一、待確認事項清單

以下事項需在 Phase 1 訪談後確認：

- [ ] **委外加工（Subcontracting）**：是否有加工產品線的委外加工需求？
- [ ] **人工成本核算**：是否需要在 ERPNext 中核算人工工時成本？
- [ ] **批號格式**：原物料批號 vs 成品批號的命名規則？
- [ ] **多幣種**：是否有進口原物料（外幣採購）？
- [ ] **條碼/標籤設備**：倉庫是否配備條碼掃描器？標籤印表機？
- [ ] **移動裝置操作**：倉管/生產人員是否用手機/平板操作 ERP？
- [ ] **ERP 整合**：是否需要與財務軟體（如用友、金蝶）或客戶/供應商系統對接？
- [ ] **生產排程展示**：是否需要生產計劃看板（甘特圖或白板式介面）？
- [ ] **品質管理**：收料是否需要 QC 檢驗流程（Quality Inspection）？
- [ ] **電纜成品計算**：電纜按米計量，是否有特殊計量換算需求？
